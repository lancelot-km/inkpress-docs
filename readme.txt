  InkPress — Customer Setup & Operations Guide
═══════════════════════════════════════════════════════════════════════════════

  An encrypted, self-hosted publishing studio sold as a license.
  You buy a license, install on your own VPS, and publish from your own domain.

  Latest version of this document:    https://inkpress-docs.zorohq.com/readme.txt

═══════════════════════════════════════════════════════════════════════════════
  1. WHAT YOU GET
═══════════════════════════════════════════════════════════════════════════════

  Four containers running on your own VPS, all on loopback ports:

    inkpress-storage         Postgres 17           (private, no port)
    inkpress-backend         Spring Boot JSON API  127.0.0.1:18080
    inkpress-admin-frontend  Angular Studio SPA    127.0.0.1:18082
    inkpress-public-frontend Angular reader SPA    127.0.0.1:18083

  Your reverse proxy (Caddy recommended) terminates TLS and routes three
  public hostnames to the loopback ports above. See Section 5.

  Everything runs on a single host. The DB, the API, the admin UI, and the
  public site are all on your box. We never see your content, your subscribers,
  or your keys.


═══════════════════════════════════════════════════════════════════════════════
  2. WHAT YOU NEED BEFORE STARTING
═══════════════════════════════════════════════════════════════════════════════

  • A Linux VPS with Docker 24+ and Docker Compose v2.
      Recommended providers: Hetzner CX22 (€4/mo), Vultr ($5/mo),
      DigitalOcean Basic ($4/mo), OVH Eco (€3/mo).
      1 GB RAM minimum, 2 GB recommended. 10 GB disk.

  • A domain you control with DNS pointing at your VPS IP.

  • Your InkPress license key (format: IPRS-XXXX-XXXX-XXXX-LIFE).
      Get yours at https://inkpress-app.zorohq.com/pricing.
      Pay-what-you-want — set $0 if you can't afford it and the same key
      lands in your account. One free per person; pay anything for extras.

  • An admin email + password — these are the writer's login credentials
    for the studio. You set them at install time; can be changed later.

  3. ONE-LINE INSTALL
═══════════════════════════════════════════════════════════════════════════════

  SSH into your VPS as root and run:

      curl -fsSL https://inkpress.zorohq.com/install.sh | \
        INKPRESS_DOMAIN=customer.com \
        INKPRESS_LICENSE_KEY=IPRS-XXXX-XXXX-XXXX-LIFE \
        ADMIN_BOOTSTRAP_EMAIL=you@customer.com \
        ADMIN_BOOTSTRAP_PASSWORD='change-me-on-first-login' \
        sh

  What the script does, in order:

    1. Checks Docker is installed and Compose v2 is available.
    2. Creates /opt/inkpress/ as the install directory.
    3. Downloads the latest docker-compose.yml from
       https://raw.githubusercontent.com/lancelot-km/inkpress-install/main/compose/docker-compose.yml
    4. Generates a random Postgres password and writes /opt/inkpress/.env
       with mode 600.
    5. Runs `docker login registry.zorohq.com -u customer -p <your-license-key>`
       — this validates your license against our license server. Bad key = the
       script exits with an error.
    6. `docker compose pull` — pulls the 4 customer-side container images
       from registry.zorohq.com.
    7. `docker compose up -d` — starts the stack in the background.
    8. Polls /actuator/health/liveness on the backend until it's healthy.

  When the script finishes you'll see the loopback ports printed. Now you need
  a reverse proxy — Section 5.


═══════════════════════════════════════════════════════════════════════════════
  4. ENV FILE REFERENCE
═══════════════════════════════════════════════════════════════════════════════

  /opt/inkpress/.env is generated by install.sh. You can edit it later, then
  `docker compose --env-file .env -f infra/docker-compose.yml up -d` to apply.

  Required:
    INKPRESS_DOMAIN           Your domain (used by the backend for canonical URLs)
    INKPRESS_LICENSE_KEY      Your license key
    ADMIN_BOOTSTRAP_EMAIL     First admin user (only used on initial bootstrap)
    ADMIN_BOOTSTRAP_PASSWORD  First admin user's password (initial bootstrap only)
    POSTGRES_PASSWORD         Random hex written by install.sh — DO NOT lose it

  Optional / overrides:
    REGISTRY                  Default: registry.zorohq.com
    IMAGE_TAG                 Default: latest. Pin to v0.X.Y for reproducible deploys.
    POSTGRES_DB               Default: inkpress
    POSTGRES_USER             Default: inkpress
    CONTROL_PLANE_BASE_URL    Default: https://inkpress-license-stg.zorohq.com
                              (the URL your backend uses to refresh license
                              entitlement once per day)

  5. REVERSE PROXY SETUP (CADDY)
═══════════════════════════════════════════════════════════════════════════════

  Install Caddy on your VPS:

      sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
      curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | \
        sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
      curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | \
        sudo tee /etc/apt/sources.list.d/caddy-stable.list
      sudo apt update && sudo apt install caddy

  Drop this into /etc/caddy/Caddyfile (change "customer.com" to your domain):

      customer.com {
        encode zstd gzip
        reverse_proxy 127.0.0.1:18083
        header /index.html Cache-Control "no-cache, no-store, must-revalidate"
      }

      admin.customer.com {
        encode zstd gzip
        reverse_proxy 127.0.0.1:18082
        header /index.html Cache-Control "no-cache, no-store, must-revalidate"
      }

      api.customer.com {
        encode zstd gzip
        reverse_proxy 127.0.0.1:18080 {
          header_up Host {host}
          header_up X-Forwarded-Proto https
        }
      }

  Reload:
      sudo systemctl reload caddy

  Caddy auto-provisions TLS certificates from Let's Encrypt on first request.
  You need port 80 + 443 open in your firewall.

  6. FIRST LOGIN
═══════════════════════════════════════════════════════════════════════════════

  Open https://admin.customer.com in a browser and sign in with the email +
  password you supplied to install.sh.

  Inside the Studio you can:
    - Set the publication name + tagline (Settings → Masthead)
    - Write your first post (Editor)
    - Add subscribers (Subscribers → Import or bulk add)
    - Configure the theme + colors (Settings → Theme)
    - View license status (License)
    - Configure backups (Backup & export — coming)
    - Manage device keys (Keys & devices — v2)

  Visit https://customer.com to see the public reader. Empty until you publish.

  7. UPDATES
═══════════════════════════════════════════════════════════════════════════════

  We push a new version to registry.zorohq.com after our team has tested it
  on staging and manually promoted via the deploy pipeline. Your VPS will only
  pull what's tagged :latest (= the promoted version).

  To update on your VPS:

      curl -fsSL https://inkpress.zorohq.com/update.sh | sh

  Or pin to a specific version:

      curl -fsSL https://inkpress.zorohq.com/update.sh | sh -s -- v0.2.3

  The script will:
    1. Read /opt/inkpress/.env
    2. Update IMAGE_TAG (to "latest" by default, or your pinned version)
    3. docker compose pull
    4. docker compose up -d  (rolling restart — usually 5-10s downtime)

  Rollback to a previous version:

      curl -fsSL https://inkpress.zorohq.com/update.sh | sh -s -- v0.2.2

  8. BACKUPS
═══════════════════════════════════════════════════════════════════════════════

  Two things to back up: the Postgres data volume and the media files volume.

  Manual backup:

      docker run --rm \
        -v inkpress_pg_data:/data \
        -v $(pwd):/backup \
        alpine tar czf /backup/inkpress-pg-$(date +%Y%m%d).tar.gz /data

      docker run --rm \
        -v inkpress_media:/data \
        -v $(pwd):/backup \
        alpine tar czf /backup/inkpress-media-$(date +%Y%m%d).tar.gz /data

  Restore:

      docker compose down
      docker volume rm inkpress_pg_data inkpress_media
      docker volume create inkpress_pg_data
      docker volume create inkpress_media
      docker run --rm -v inkpress_pg_data:/data -v $(pwd):/backup \
        alpine tar xzf /backup/inkpress-pg-YYYYMMDD.tar.gz -C /
      docker run --rm -v inkpress_media:/data -v $(pwd):/backup \
        alpine tar xzf /backup/inkpress-media-YYYYMMDD.tar.gz -C /
      docker compose up -d

  Recommended: schedule the backup snippet as a cron job + rclone to off-site
  storage (Cloudflare R2, IPFS pinata, S3, etc).


═══════════════════════════════════════════════════════════════════════════════
  9. TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

  Containers up?
      docker ps --filter name=inkpress

  Backend logs?
      docker logs inkpress-backend --tail 100 -f

  Backend health endpoint?
      curl http://127.0.0.1:18080/actuator/health/liveness

  Postgres reachable from backend?
      docker exec inkpress-backend sh -c 'echo > /dev/tcp/inkpress-storage/5432 && echo OK'

  License check failing?
      docker logs inkpress-backend 2>&1 | grep -i license

  Reset admin password (lost it)?
      docker exec -it inkpress-storage psql -U inkpress -d inkpress -c \
        "DELETE FROM admin_user WHERE email='you@customer.com';"
      docker restart inkpress-backend
      # Restart re-bootstraps the admin with current env values

  Disk filling up?
      docker system prune -a --volumes  # WARNING: deletes unused images + volumes
      # OR target just images:
      docker image prune -a

  10. SECURITY NOTES
═══════════════════════════════════════════════════════════════════════════════

  • All container ports bind to 127.0.0.1 only. Only Caddy (on the host) is
    publicly reachable.
  • Subscriber email addresses are stored SHA-256 hashed; the originals are
    never written to disk.
  • Drafts and the post body are stored in the Postgres volume in plaintext
    — the encryption story we plan for v2 wraps the DB volume, not the
    individual fields.
  • The Studio (admin.customer.com) accepts HTTP basic-auth-style login.
    Use a strong password. Per-device ed25519 pairing arrives in v2.
  • The license check is one signed HTTP request per day from your backend
    to inkpress-license-stg.zorohq.com. No content, no telemetry.
  • There is no telemetry script on the public reader. There is no ad pixel.
    There never will be.
